A-Star: Improving Dijkstra's algorithm

Notes based on this blog post

  • Dijkstra is "naive" about expansion
  • A* (A-star) prefers expansions that get us closer to the goal:
    • A "Heuristic Function" assigns an extra cost to a possible expansion node:
      • $H(x_n,y_n) = \sqrt{(x_g-x_n)^2+(y_g-y_n)^2}\leftarrow$ Euclidean Distance, OR
      • $H(x_n,y_n) = \left|x_g-x_n\right|+\left|y_g-y_n\right|\leftarrow$ Manhattan Distance

A-Star: Improving Dijkstra's algorithm

Notes based on this blog post

A-Star Can then be implemented as:

  • For each node, n, in the graph:
    • n.distance = Infinity, n.g = Infinity
  • Create an empty list
    • start.distance = 0
    • start.f=H(start)
    • add start to list
  • While list not empty:
    • Let current = node in the list with the smallest f value, remove current from list
    • If (current == goal) return Success
    • For each node, n that is adjacent to current:
      • if n.g > (current.g + cost of edge from n to current
        • n.g= current.g+ cost of edge from n to current
        • n.f = n.g + H(n)
        • n.parent = current
        • Add n to list if it is not there already

Class Exercise: Follow a path generated by Dijkstra (or A star)!

The Week 11 starter controller is "dumb" with respect to Romi motion, however. Your task is to modify its FSM so that the romi follows the path by subsequently using each point as a goal location for feedback controllers on distance and relative yaw angle.
